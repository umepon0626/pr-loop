# 要件定義書

## 概要

GitHub PRの自動修正ループシステムは、既存のGemini Code AssistによるAIレビューコメントを監視し、そのレビューコメントに基づいてKiroのようなIDE内蔵型エージェントが自動修正を行い、複数回のレビュー・修正サイクルを回すCIツールです。これにより、開発者の手動レビュー前にコード品質を自動的に向上させることを目指します。

## 要件

### 要件1：自動修正機能

**ユーザーストーリー:** 開発者として、Gemini Code AssistのAIレビューコメントが自動的にコード修正をトリガーすることで、一般的な問題が手動介入なしに解決されることを望む。

#### 受入基準

1. Gemini Code AssistのAIレビューコメントが投稿された時、システムは実行可能な提案についてコメントを解析すること
2. 実行可能な提案が特定された時、システムはIDEエージェントをトリガーして対応するコード変更を行うこと
3. コード変更が行われた時、システムは説明的なメッセージでその変更をコミットすること
4. コード修正が完了した時、システムは対応するレビューコメントをresolveすること
5. 提案が不明確または矛盾している場合、システムは自動修正をスキップし、手動レビューのためにフラグを立てること

### 要件2：ループ機能

**ユーザーストーリー:** 開発者として、レビュー・修正ループが複数回実行されることで、反復的な改善が自動的に行われることを望む。

#### 受入基準

1. コード修正がコミットされた時、システムはGemini Code Assistに新しいレビューを要求すること
2. レビューサイクルが完了した時、システムは最大ループ回数（デフォルト3回）に達したかチェックすること
3. 最大ループ回数に達したか、新しい問題が見つからなかった時、システムはループを停止すること
4. 各ループ反復が開始される時、システムは適切な遅延（デフォルト2分）を待ってから進行すること

### 要件3：進捗表示機能

**ユーザーストーリー:** 開発者として、自動レビューループの履歴とステータスを確認できることで、どのような変更が行われたかを理解したい。

#### 受入基準

1. レビューループが開始された時、システムはPRにステータスコメントを作成すること
2. 各反復が完了した時、システムは進捗でステータスコメントを更新すること
3. ループが完了した時、システムは行われたすべての変更の要約を提供すること
4. PRを表示する時、すべての自動コミットは明確にラベル付けされ、グループ化されること
